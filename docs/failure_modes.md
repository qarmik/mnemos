## FM-94+ — Open

FM-94 and beyond require real human sessions to discover.

**FM-94 candidate (identified from simulation analysis):**
Preference signal accumulation is slower than needed when a user asks varied questions about the same underlying topic. The topic key clustering (FM-88) uses content-word overlap, which fragments when a user paraphrases. A semantic similarity component may be needed alongside lexical overlap.

This remains a candidate until observed in a real session.

---

## On the register staying open

FM-50 was specifically designed to prevent belief in completeness. The adversarial layer (L15) runs continuously and probes for new failure modes as part of normal operation. The FM register is append-only and never declared complete.

The pattern so far: simulation finds ANNOYING failures. Real sessions will find a different class — failures that scripted queries cannot generate because they require genuine ambiguity, emotional subtext, and multi-intent phrasing.

FM-94 is waiting in the first real session.
